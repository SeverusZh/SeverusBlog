签名部分优化-ENCODE-01

解析 encode_sig函数

在完成了GAUSS函数相关优化后，我们来到了签名部分的最后一个优化点——encode_sig函数，签名部分其他的函数或者宏几乎都是串行执行的指令且没有经过循环过程，并不适合开展并行化。

接下来我们开始解析 encode_sig函数。

逐行解析

  1. 参数声明：

    • 功能：定义函数，接受多项式的对数 logn、输出签名缓冲区 sig及其长度 sig_len、盐值 salt及其长度 salt_len，以及签名多项式 s1。

  2. 计算多项式项数：

    • 含义：计算多项式的项数 n为 2^logn。

  3. 设置低位位数：

    • 含义：根据 logn确定编码时低位的位数。若 logn为10（1024项），low设为6，否则为5。

  4. 初始化缓冲区指针：

    • 含义：将输出缓冲区 sig转换为 uint8_t指针 buf，并记录剩余长度 buf_len。

  5. 检查最小缓冲区大小：

    • 含义：确保缓冲区足够容纳盐值和固定部分。若不足，返回失败。

  6. 复制盐值到缓冲区：

    • 含义：将盐值复制到缓冲区，调整指针和剩余长度。

  7. 编码符号位：

    • 含义：提取 s1数组中每个元素的符号位（最高位），每8个符号位组合成一个字节，存入缓冲区。

  8. 调整缓冲区指针：

    • 含义：跳过符号部分，更新剩余长度。

  9. 处理固定大小的低位部分：

    • 含义：将每个元素的低 low位（绝对值）打包成字节。tbmask用于处理符号，将负数转换为正数，low_mask提取低位。

变量列表及含义

变量名 类型 含义
logn unsigned 多项式的对数次数，n=2^logn。
sig void* 输出签名的缓冲区指针。
sig_len size_t 输出缓冲区的总长度。
salt const uint8_t* 盐值的指针。
salt_len size_t 盐值的长度。
s1 const int16_t* 待编码的签名多项式系数数组。
n size_t 多项式的项数，等于2^logn。
low int 编码时低位的位数，logn=10时为6，否则为5。
buf uint8_t* 当前写入位置的指针，指向输出缓冲区。
buf_len size_t 输出缓冲区剩余可用长度。
low_mask uint32_t 低位掩码，用于提取每个系数的低 low位，如0x1F或0x3F。
u, v, vv size_t 循环变量，用于遍历多项式的系数和位操作。
x uint64_t 临时变量，用于存储固定部分打包后的位数据。
i int 循环变量，处理字节内的位偏移。
acc uint32_t 累积变量，用于存储可变长度编码的位流。
acc_len int 当前累积的位数。
k int 当前元素的高位部分的位数，用于生成一元码。
w uint32_t 临时变量，存储当前处理的 s1元素的值（转换为无符号）。
tbmask 宏或函数 处理符号位的函数，将负数转换为正数（如符号位扩展后异或）。
